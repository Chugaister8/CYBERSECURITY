# 4.3 Галузь знань: Безпека компонентів

Галузь знань «Безпека компонентів» зосереджена на проектуванні, закупівлі, тестуванні, аналізі та обслуговуванні компонентів, що інтегруються в більші системи.

Безпека системи частково залежить від безпеки її компонентів. Безпека компонента залежить від того, як він спроектований, виготовлений, закуплений, протестований, підключений до інших компонентів, використовується та підтримується. Ця галузь знань передусім стосується аспектів безпеки проектування, виготовлення, закупівлі, тестування та аналізу компонентів. Разом із KA «Безпека з'єднань» та KA «Безпека систем», KA «Безпека компонентів» розглядає проблеми безпеки при підключенні компонентів та їхньому використанні в складі більших систем.

## 4.3.1 Одиниці знань та теми

Наведена нижче таблиця перелічує основні елементи, одиниці знань та теми галузі знань «Безпека компонентів».

---

**БЕЗПЕКА КОМПОНЕНТІВ**

**Основні елементи:**
- Вразливості компонентів системи,
- Lifecycle компонента,
- Принципи захищеного проектування компонентів,
- Безпека управління ланцюгом постачання,
- Тестування безпеки,
- Зворотна інженерія.

---

| Одиниці знань | Теми | Опис / Навчальні рекомендації |
|---|---|---|
| **Проектування компонентів** *[Див. також KA «Безпека програмного забезпечення» щодо пов'язаного змісту, с. 23.]* | | Ця одиниця знань знайомить із принципами та техніками проектування, що підвищують безпеку компонентів. |
| | Безпека проектування компонентів | Ця тема охоплює загрози безпеці артефактів проектування компонентів (наприклад, схем, netlist та масок), такі як апаратні трояни, піратство інтелектуальної власності, зворотна інженерія, фальсифікація, аналіз побічних каналів та підробка. Вона також знайомить із техніками захисту компонентів від несанкціонованого доступу та використання. |
| | Принципи захищеного проектування компонентів | Ця тема охоплює такі принципи, як встановлення обґрунтованої політики безпеки, розгляд безпеки як невід'ємної частини проектування системи, платформи довірених обчислень, ланцюжок довіри, зниження ризику, багаторівнева безпека, простота проектування, мінімізація елементів системи, яким надається довіра, та уникнення зайвих механізмів безпеки. |
| | Ідентифікація компонентів | Ця тема охоплює такі техніки, як водяні знаки, дактилоскопія, метрування, зашифровані ідентифікатори та фізично нечіткі функції (PUF) для захисту компонентів від крадіжки інтелектуальної власності та забезпечення автентичності компонентів. |
| | Техніки захисту від зворотної інженерії | Ця тема охоплює такі техніки, як обфускація дизайну та камуфлювання, що ускладнюють зворотну інженерію дизайнів та реалізацій компонентів. |
| | Протидія атакам побічних каналів | Ця тема охоплює техніки захисту від атак побічних каналів, спрямованих передусім на криптографічні алгоритми. Захисні техніки включають зниження витоку інформації, введення шуму, часте оновлення ключів, фізичні випадкові функції та захищені ланцюжки сканування. |
| | Технології захисту від фальсифікації | Ця тема охоплює техніки підвищення стійкості компонентів до фізичних та електронних атак, включаючи методи фізичного захисту, системи виявлення фальсифікації та системи реагування на фальсифікацію. |
| **Закупівля компонентів** | | Ця одиниця знань описує техніки забезпечення безпеки компонентів системи впродовж усього процесу закупівлі. |
| | Ризики ланцюга постачання | Ця тема описує загрози та ризики безпеці як апаратного, так і програмного забезпечення при закупівлі компонентів. |
| | Безпека ланцюга постачання | Ця тема описує стратегії, такі як фізична безпека, роздільне виробництво, відстежуваність, перевірка та інспекція вантажів для виявлення та запобігання компрометації безпеки компонентів у процесі закупівлі. |
| | Перевірка постачальників | Ця тема включає такі стратегії, як акредитація постачальників для встановлення довірених постачальників і перевізників компонентів. |
| **Тестування компонентів** *[Див. також KA «Безпека програмного забезпечення» щодо пов'язаного змісту, с. 23.]* | | Ця одиниця знань знайомить із техніками модульного тестування та описує інструменти і техніки, що використовуються для тестування властивостей безпеки компонента. |
| | Принципи модульного тестування | Ця тема описує інструменти та техніки модульного тестування на відміну від тестування на рівні системи. |
| | Тестування безпеки | Ця тема описує інструменти та техніки, такі як фаззинг-тестування, для перевірки властивостей безпеки компонента поза межами його функціональної коректності. |
| **Зворотна інженерія компонентів** | | Ця одиниця знань описує техніки виявлення дизайну та функціональності компонента за неповної інформації. |
| | Зворотна інженерія дизайну | Ця тема описує інструменти та техніки виявлення дизайну компонента на певному рівні абстракції. |
| | Зворотна інженерія апаратного забезпечення | Ця тема описує інструменти та техніки виявлення функціональності та інших властивостей апаратного забезпечення компонента, наприклад функцій інтегральної схеми. |
| | Зворотна інженерія програмного забезпечення | Ця тема описує інструменти та техніки, такі як статичний та динамічний аналіз, для виявлення функціональності та властивостей програмного забезпечення компонента. |

## 4.3.2 Основні елементи та результати навчання

Від студентів вимагається демонструвати компетентність у кожному з основних понять через досягнення результатів навчання. Як правило, результати навчання лежать у межах рівнів розуміння та застосування за переглянутою таксономією Блума (http://ccecc.acm.org/assessment/blooms).

| Основні елементи | Результати навчання |
|---|---|
| Вразливості компонентів системи | Пояснити, як безпека компонентів системи може впливати на безпеку системи загалом. |
| | Описати способи, якими може бути скомпрометована конфіденційність дизайну компонента. |
| | Описати способи отримання інформації про функціональність компонента за обмеженої інформації про його дизайн та реалізацію. |
| Lifecycle компонента | Перелічити фази lifecycle компонента. |
| Принципи захищеного проектування компонентів | Перелічити артефакти проектування компонентів, які можуть потребувати захисту. |
| | Навести приклади кількох принципів захищеного проектування компонентів та пояснити, як кожен із них захищає безпеку компонентів. |
| | Описати кілька технік захисту елементів дизайну інтегральної схеми. |
| Безпека управління ланцюгом постачання | Перелічити поширені точки вразливості в ланцюзі постачання компонента. |
| | Описати ризики безпеки в ланцюзі постачання компонентів. |
| | Описати способи зниження ризиків ланцюга постачання. |
| Тестування безпеки | Розмежувати модульне та системне тестування. |
| | Перелічити кілька технік тестування властивостей безпеки компонента. |
| Зворотна інженерія | Перелічити причини, з яких хтось може застосовувати зворотну інженерію до компонента. |
| | Пояснити різницю між статичним та динамічним аналізом при зворотній інженерії програмного забезпечення. |
| | Описати техніку зворотної інженерії функціональності інтегральної схеми. |
