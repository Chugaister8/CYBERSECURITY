# Розділ 4: Зміст навчального фреймворку з кібербезпеки

Навчальний зміст було зібрано та синтезовано з різноманітних джерел, зокрема (без урахування пріоритету): ACM/IEEE CS2013; ACM/IEEE IT2017; Центри академічної досконалості Агентства національної безпеки та Міністерства внутрішньої безпеки США (CAE); (ISC)²; фреймворки розвитку кадрів, такі як Фреймворк кадрів у сфері кібербезпеки Національної ініціативи США з освіти в галузі кібербезпеки (NCWF); Global IT Skills Framework for the Information Age (SFIA); зразки курсів, спонсоровані Intel University Programs Office; U.S. National Science Foundation; U.K. Government Communications Headquarters (GCHQ); галузеві робочі групи та інші джерела, надані спільнотою зацікавлених сторін.

Розділи цієї глави надають огляд навчального змісту для кожної галузі знань. Таблиця для кожної галузі знань перелічує основні елементи, одиниці знань та теми в межах кожної одиниці знань. У багатьох випадках включено конкретні навчальні рекомендації щодо охоплення тем. Для уточнення одиниць знань і тем JTF скликала профільних експертів у Робочих групах галузей знань (KAWG). Члени KAWG перераховані за галузями знань у Додатку A.

Як описано вище, основні елементи у галузях знань відображають ту компетентність у кібербезпеці, якої кожен студент має досягти незалежно від напрямку програми. Основні елементи наведені в кожному розділі KA та представлені разом у Додатку B.

**Примітка.** Деякі одиниці знань і теми в галузях знань здаються надмірно повторюваними. Це свідоме повторення, що служить як для забезпечення конкретності охоплення в кожній окремій галузі знань, так і для підкреслення важливості цих основних одиниць знань і тем у сукупному домені знань дисципліни кібербезпеки.

У таблицях галузей знань містяться перехресні посилання на інші таблиці галузей знань, що містять важливу пов'язану інформацію. Ці перехресні посилання розміщені в крайньому лівому стовпці під «Одиницями знань» і виділені курсивом у квадратних дужках [ ].

Див. Додаток C для огляду зразків, що відображають галузі знань та одиниці знань на різні типи навчальних програм. Навчальні зразки демонструють, як навчальні програми конкретних установ охоплюють основні елементи галузей знань і деяку підмножину одиниць знань. Зразки розміщені на веб-сайті залучення спільноти (http://cybered.acm.org/), щоб показати, як зміст з кібербезпеки може бути організований різноманітними способами.

Інформація в таблицях надає рекомендації розробникам цілої навчальної програми або силабусу курсу. Ці розробники повинні наповнити теми конкретним матеріалом, який їхній курс має охоплювати. Наприклад, опис/навчальні рекомендації для теми «Логічний контроль доступу до даних» в одиниці знань «Контроль доступу» галузі знань «Безпека даних» перелічують кілька типів засобів контролю, зокрема списки контролю доступу, обов'язковий контроль доступу тощо. Курс із безпеки мобільних пристроїв матиме в силабусі засоби контролю доступу для Android, навіть якщо це явно не вказано в стовпці «Опис/Навчальні рекомендації». На момент написання цього документа система Android не надає контролю доступу на основі атрибутів, тому силабус цього курсу наразі його не включатиме. Аналогічно, силабус з сучасної криптографії включатиме «Режими роботи блокових шифрів» (обговорення/навчальні рекомендації в темі «Симетричні шифри (з секретним ключем)» одиниці знань «Криптографія» в галузі знань «Безпека даних»), такі як GCM, навіть якщо обговорення/навчальні рекомендації явно не згадують цей режим. Ці два приклади підкреслюють, що навчальні рекомендації представляють теми, які навчальна програма може включати. Конкретний зміст цих тем залишається на розсуд розробника навчальної програми, оскільки він знає цілі курсу та потреби студентів і може адаптувати охоплення кожної теми для досягнення цих цілей і задоволення цих потреб.

## 4.1 Галузь знань: Безпека даних

Галузь знань «Безпека даних» зосереджена на захисті даних у стані спокою, під час обробки та під час передавання. Ця галузь знань потребує застосування математичних і аналітичних алгоритмів для повноцінної реалізації.

### 4.1.1 Одиниці знань та теми

Наведена нижче таблиця перелічує основні елементи, одиниці знань та теми галузі знань «Безпека даних».

---

**БЕЗПЕКА ДАНИХ**

**Основні елементи:**
- Базові концепції криптографії,
- Цифрова криміналістика,
- Наскрізні захищені комунікації,
- Цілісність даних та автентифікація,
- Безпека зберігання інформації.

| | Законодавство у сфері безпеки даних | Ця тема знайомить із правовими аспектами безпеки даних, законами та політиками, що регулюють дані (наприклад, HIPAA). Вона також надає вступ до інших правових тем у галузі знань «Організаційна безпека». |

### 4.1.2 Основні елементи та результати навчання

Від студентів вимагається демонструвати компетентність у кожному з основних понять через досягнення результатів навчання. Як правило, результати навчання лежать у межах рівнів розуміння та застосування за переглянутою таксономією Блума (http://ccecc.acm.org/assessment/blooms).

| Основні елементи | Результати навчання |
|---|---|
| Базові концепції криптографії | Описати призначення криптографії та перелічити способи її використання в передаванні даних. |
| | Описати такі терміни: шифр, криптоаналіз, криптографічний алгоритм та криптологія, а також описати два основні методи (шифри) перетворення відкритого тексту на шифротекст. |
| | Пояснити, як інфраструктура відкритих ключів підтримує цифровий підпис і шифрування, та обговорити обмеження/вразливості. |
| | Обговорити небезпеки самостійного винаходу власних криптографічних методів. |
| | Описати, які криптографічні протоколи, інструменти та техніки є доцільними для конкретної ситуації. |
| Наскрізні захищені комунікації *[Див. також KA «Безпека з'єднань» щодо пов'язаного змісту, с. 32.]* | Пояснити цілі наскрізної безпеки даних. |
| Цифрова криміналістика | Описати, що таке цифрове розслідування, джерела цифрових доказів та обмеження криміналістики. |
| | Порівняти та зіставити різноманітні криміналістичні інструменти. |
| Цілісність даних та автентифікація | Пояснити концепції автентифікації, авторизації, контролю доступу та цілісності даних. |
| | Пояснити різні техніки автентифікації та їхні переваги й недоліки. |
| | Пояснити можливі атаки на паролі. |
| Знищення даних | Описати різні техніки знищення даних. |
