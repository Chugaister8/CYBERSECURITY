# 4.7 Галузь знань: Організаційна безпека

Галузь знань «Організаційна безпека» зосереджена на захисті організацій від кіберзагроз та управлінні ризиками для підтримки успішного виконання місії організації. Організації несуть відповідальність за задоволення потреб багатьох зацікавлених сторін, і ці потреби повинні лежати в основі кожної з наведених нижче одиниць знань.

## 4.7.1 Одиниці знань та теми

Студенти повинні вміти визначати типи законів, нормативних актів та стандартів безпеки, в межах яких функціонує організація. Державна організація має певний набір профілів безпеки, тоді як корпоративна структура має інші пріоритети. Політика безпеки повинна відповідати поточній організації та бути здатною розвиватися разом із нею. Фахівець із безпеки повинен розуміти чинні системи управління та те, як вони забезпечують дотримання вимог у відповідних бізнес-вертикалях, таких як охорона здоров'я та електронна комерція.

Наступна таблиця перелічує основні елементи, одиниці знань та теми галузі знань «Організаційна безпека». Через збіг тем слід посилатися на одиниці знань у галузі знань «Суспільна безпека».

| **Управління ризиками** | | Управління ризиками — це пошук ризиків для організаційних інформаційних активів та контроль над ними. |
| | Ідентифікація ризиків | Ідентифікація активів — це каталогізація інформаційних активів організації, таких як бази даних або апаратне забезпечення, для сприяння визначенню ризику у разі компрометації або втрати активів. Загрози включають будь-яку подію, що використовує вразливість і потенційно може спричинити збитки або шкоду для організації. Розвідка загроз (моделювання загроз) дедалі частіше використовується організаціями для підтримання обізнаності та реактивної спроможності щодо існуючих та нових загроз. |
| | Оцінювання та аналіз ризиків | Аналіз ризиків — це організаційний процес визначення можливих випадкових або навмисних збитків та їхнього врегулювання, а також проектування та реалізації процедур для мінімізації наслідків цих збитків. Він також може охоплювати аналіз загроз та розвідку загроз. |
| | Внутрішні загрози | Ця тема охоплює зловмисні поведінкові фактори людини, що можуть завдати шкоди внаслідок свідомого порушення довіри, найкращих практик використання або ненавмисної помилки. Інсайдер визначається як будь-яка особа, що має авторизований доступ до ресурсів організації, включаючи персонал, об'єкти, інформацію, обладнання, мережі та системи. Внутрішня загроза визначається як ризик того, що інсайдер використає свій авторизований доступ — навмисно або ненавмисно — для завдання шкоди своїй організації. Це може включати крадіжку пропрієтарної інформації та технологій; пошкодження об'єктів, систем або обладнання компанії; фактичну або потенційну шкоду співробітникам; або інші дії, що перешкоджатимуть нормальній діяльності компанії. Ця тема охоплює поведінку мотив-засіб-можливість: фактори мотивації та дисципліни, підзвітність, обізнаність та контроль якості. ФБР розробило матеріали з індикаторами, корисними для ідентифікації потенційних ризиків внутрішніх загроз. |
| | Моделі та методології вимірювання та оцінювання ризиків | Моделі ризиків використовуються для пояснення того, як активи зазнають ризику. Крім того, існує ряд прийнятих у галузі методологій для вимірювання, оцінювання та комунікації ризиків зацікавленим сторонам. Ця тема включає як кількісні, так і якісні підходи до оцінювання ризиків, застосування моделей та методів для різних бізнес-контекстів (наприклад, HIPAA для медичних закладів). Серед корисних інструментів можна виділити самооцінку Cyber Resilience Review, інструмент оцінювання кібербезпеки (CSET), а також інструмент оцінювання ризику безпеки від HSS. |
| | Контроль ризиків | Контроль ризиків визначається як дія зі зменшення наслідків кіберінциденту і, як результат, зменшення рівня ризику. Кожен підхід повинен включати засоби комунікації ризику особам, що приймають рішення, включаючи залишковий ризик. Теми, що охоплюються, повинні включати оцінювання та ранжування ризиків та категорії «уникнути, зменшити, передати, прийняти». Навчальний зміст повинен включати широко використовувані методології контролю ризиків, доступні для ознайомлення та практики. |
| **Безпекове управління та політика** *[Див. також KA «Суспільна безпека» щодо пов'язаного змісту, с. 62.]* | | Кожна організація реагує на своє операційне середовище — внутрішнє та зовнішнє — через політику та управління. Управління — це відповідальність вищого керівництва організації за забезпечення ефективної реалізації стратегічного планування, управління ризиками та регуляторної відповідності, як правило, за допомогою комплексних управлінських політик, планів, програм та бюджетного контролю для захисту інформації організації. Реалізація безпекового управління та політики повинна бути вписана в рамки глобальних, національних та місцевих законів, нормативних актів та стандартів. Ця одиниця знань зосереджена на розумінні циклу розроблення політики безпеки — від початкового дослідження до реалізації та підтримання, — а також на ознайомленні з реальними прикладами політик і практик безпеки. |
| | Організаційний контекст | Багато факторів впливають на те, як безпека реалізується в організаціях. Ці контексти є критично важливими при розробленні навчальної програми та повинні інформувати весь процес. Ця тема охоплює те, як внутрішні та зовнішні контекстуальні відмінності суттєво впливають на охоплення політики, нормативних актів та статутів (або юрисдикцій). Також повинні бути оцінені питання та проблеми, специфічні для конкретного місця або країни. Застосовні стандарти та рекомендації щодо відповідності вимогам галузі/сектору також підлягають оцінюванню. Відмінності між державними та приватними організаціями є значущим фактором, як і необхідність включення міжнародних аспектів, включаючи, але не обмежуючись, обмеженнями на імпорт/експорт. Крім того, існують суттєві відмінності між організаціями в різних бізнес-вертикальних галузях, таких як енергетика та сільське господарство. |
| | Конфіденційність *[Див. також KA «Безпека даних», с. 16, KA «Безпека людини», с. 44, та KA «Суспільна безпека», с. 62, щодо пов'язаного змісту.]* | Конфіденційність — це концепція з культурними та національними варіаціями у своєму визначенні. У своїй основі конфіденційність базується на праві бути забутим та різних рівнях вибору та згоди щодо збору, використання та розповсюдження інформації про особу. Ця тема розглядає соціальні та локалізовані варіації конфіденційності. Слід розглянути юрисдикційні варіації у визначеннях конфіденційності. Також повинні бути розглянуті відносини між особистими, організаційними або державними вимогами щодо конфіденційності. Вплив налаштувань конфіденційності в нових інструментах/програмному забезпеченні та визначення потреби в інструментах та техніках для охоплення в більшості галузей. Додатково слід приділити увагу конфіденційності в контексті захисту прав споживачів та регулювання охорони здоров'я. Організації з міжнародною діяльністю повинні враховувати варіації законів, нормативних актів та стандартів у сфері конфіденційності в юрисдикціях, в яких вони функціонують. |
| | Закони, етика та відповідність вимогам *[Див. також KA «Суспільна безпека» щодо пов'язаного змісту, с. 62.]* | Закони, нормативні акти, стандарти, а також етичні цінності випливають із соціального контексту та того, як організації відповідають вимогам щодо їх дотримання. Ця тема включає те, як закони та технології перетинаються в контексті судових структур, що існують — міжнародних, національних та місцевих — коли організації захищають інформаційні системи від кібератак. Навчання з етики також повинно бути складовою. Слід розглянути професійні кодекси поведінки та етичні стандарти. Зусилля з дотримання вимог повинні включати заходи щодо відповідності законам, нормативним актам та стандартам, а також включати вимоги щодо сповіщення про порушення з боку державних, національних та міжнародних регуляторних органів. Приклади міжнародних законів та стандартів включають GDPR та ISO/IEC 27000 тощо. Важливі національні закони для організацій США включають HIPAA, Sarbanes-Oxley, GLBA тощо. |
| | Управління безпекою | Принципи корпоративного управління застосовні до функції інформаційної безпеки. Управління — це відповідальність вищого керівництва організації за забезпечення ефективної реалізації стратегічного планування, управління ризиками та регуляторної відповідності, як правило, за допомогою комплексних управлінських політик, планів, програм та бюджетного контролю для захисту інформації організації. Ця тема повинна вписувати реалізацію безпекового управління та політики в рамки глобальних, національних та місцевих законів, нормативних актів та стандартів, а програми навчання повинні прагнути донести концепції чітко та на якісних прикладах. |
| | Комунікація на рівні керівництва та ради директорів | Надання інформації виконавчим директорам та зовнішнім особам, що приймають рішення, — це критично важлива навичка для керівників у сфері інформаційної безпеки. Ця тема включає навички комунікації, що викладаються та відпрацьовуються на репетиціях із критичним аналізом та змістовним зворотним зв'язком. |
| | Управлінська політика | Організаційні рекомендації, що диктують певну поведінку в межах організації. Зміст цієї теми повинен прагнути донести концепції чітко та на якісних прикладах, включаючи політику програми безпеки, проблемно-орієнтовану політику та системно-специфічну політику відповідно до NIST SP 800-12 Rev 1. Це також повинно охоплювати розуміння циклу розроблення політики безпеки — від початкового дослідження до реалізації та підтримання, — а також ознайомлення з реальними прикладами політик і практик безпеки. |
| **Аналітичні інструменти** | | Ця одиниця знань є набором технік використання аналітики даних для розпізнавання, блокування, відведення та реагування на кібератаки. Моніторинг мережевої активності в реальному часі забезпечує гнучке прийняття рішень, виявлення підозрілих зловмисних дій, використання інформаційних панелей візуалізації в реальному часі та застосування набору апаратних і програмних засобів для управління виявленою підозрілою активністю. |
| | Показники ефективності (метрики) | Процес проектування, реалізації та управління використанням конкретних вимірювань для визначення ефективності загальної програми безпеки. Базується на метриках — терміні, що використовується для опису будь-якої детальної техніки статистичного аналізу ефективності, але тепер є загальноприйнятим синонімом вимірювання ефективності. Навчальний зміст повинен включати пояснення студентам підходів і технік для визначення та оцінювання корисності показників ефективності. |
| | Аналітика даних | Аналітика даних — це набір технік, що використовуються для маніпулювання (часто) великими обсягами даних для розпізнавання, блокування, відведення та реагування на кібератаки. Моніторинг мережевої активності в реальному часі забезпечує гнучке прийняття рішень, виявлення підозрілих зловмисних дій, використання інформаційних панелей візуалізації в реальному часі та застосування набору апаратних і програмних засобів для управління такою виявленою підозрілою активністю. Ця тема включає: визначення; відмінності між програмним забезпеченням та інструментами контролю безпеки та аналітики безпеки; типи та класифікації аналітичних інструментів та технік (з прикладами, такими як OpenSOC); збір, фільтрацію, інтеграцію та зв'язування різних типів інформації про події безпеки; принципи роботи інструментів аналітики безпеки; відносини між аналітичним програмним забезпеченням та інструментами і криміналістикою; відмінності між криміналістичними та аналітичними інструментами; мережеву криміналістику (включаючи аналіз пакетів, інструменти, Windows, Linux, UNIX, мобільні пристрої); відмінності між кіберкриміналістикою (наприклад, соціальні мережі) та мережевою криміналістикою. |
| | Розвідка у сфері безпеки | Збір, аналіз та розповсюдження інформації про безпеку, включаючи, але не обмежуючись, загрозами та можливостями зловмисників. У межах цієї теми слід розглянути інструменти та техніки, включаючи збір та агрегування даних, видобуток даних, аналітику даних, статистичний аналіз. Приклади джерел розвідки у сфері безпеки включають SIEM для внутрішніх даних та публічні й приватні розвідувальні сервіси для зовнішніх даних. Розповсюдження включає розуміння підходу Центрів обміну інформацією та аналізу (ISAC), а також організацій, таких як InfraGard. |
| **Системне адміністрування** *[Див. також KA «Безпека даних», с. 16, KA «Безпека людини», с. 44, та KA «Суспільна безпека», с. 62, щодо пов'язаного змісту.]* | | Системне адміністрування працює у фоновому режимі для конфігурування, експлуатації, обслуговування та усунення несправностей технічної системної інфраструктури, що підтримує значну частину сучасного життя. Необхідні попередні знання: базове розуміння комп'ютерних систем (Windows/Linux), мереж (модель OSI), програмного забезпечення та баз даних (Oracle/SQL). |
| | Адміністрування операційних систем | Ця тема охоплює обслуговування, надійну роботу, конфігурування та усунення несправностей технічних систем, особливо багатокористувацьких систем та серверів. Тема включає, але не обмежується: управлінням обліковими записами, адмініструванням дисків, адмініструванням системних процесів, автоматизацією системних завдань, моніторингом та оптимізацією продуктивності, адмініструванням інструментів безпеки та резервного копіювання дисків і процесів. |
| | Адміністрування баз даних | Ця тема охоплює управління та обслуговування баз даних з використанням доступного та придатного програмного забезпечення для управління системами. Тема включає, але не обмежується: встановленням та конфігуруванням серверів баз даних, створенням та маніпулюванням схемами, таблицями, індексами, представленнями, обмеженнями, збереженими процедурами, функціями, створенням і адмініструванням облікових записів користувачів, а також інструментами резервного копіювання та відновлення баз даних. Охоплення повинно включати широко використовувані технології зберігання даних, а також нові технології управління даними. |
| | Мережеве адміністрування | Мережеве адміністрування стосується встановлення та підтримки різних мережевих системних архітектур (LAN, WAN, MAN, інтранети, екстранети, демілітаризовані зони тощо) та інших систем передавання даних. Тема включає, але не обмежується: моделлю OSI, захистом мережевого трафіку та інструментами конфігурування сервісів. |
| | Хмарне адміністрування | Хмарне адміністрування стосується обслуговування та забезпечення надійного доступу до динамічного пулу конфігурованих віддалених ресурсів (наприклад, мереж, серверів, сховищ, застосунків і сервісів), що можуть бути швидко сконфігуровані, надані та звільнені з мінімальним наглядом. Тема включає, але не обмежується: конфігуруванням та розгортанням застосунків і користувачів у хмарних інфраструктурах, аналізом продуктивності, масштабуванням ресурсів, доступністю хмарних платформ, виявленням проблем безпеки та конфіденційності та пом'якшенням ризиків. |
| | Адміністрування кіберфізичних систем | Кіберфізичні системи (CPS) — це спроектовані системи, що побудовані на основі та залежать від безперервної інтеграції обчислювальних алгоритмів і фізичних компонентів. Адміністрування CPS стосується встановлення та обслуговування шляхом забезпечення безпечності, спроможності, адаптивності, масштабованості, стійкості, захисту та зручності використання. Тема включає, але не обмежується: архітектурою кіберфізичних систем, базовими стандартами комунікації (Zigbee), middleware, сервісно-орієнтованою архітектурою, інструментами підтримки управління в реальному часі та застосуванням реальних прикладів (електроенергетична мережа, ядерний об'єкт, IoT, SCADA). |
| | Зміцнення системи | Ця тема охоплює захист системи шляхом пошуку та усунення ризиків. Це може включати зміцнення або захист конфігурації, системного програмного забезпечення, мікропрограмного забезпечення та застосунків. Тема включає, але не обмежується: ідентифікацією ризиків, загроз та вразливостей у широко використовуваних системах (операційні системи, системи баз даних, мережі); визначенням та адмініструванням процедур і практик для захисту від загроз; зміцненням за допомогою відповідних інструментів (міжмережевий екран, антивірус, IDS, honeypot). |
| | Доступність | Надійна експлуатація системи вимагає, щоб усі системи підтримували цільові рівні доступності за рахунок можливості відновлення поточного стану після збоїв через резервування та резервне копіювання й відновлення. Тема включає, але не обмежується: ідентифікацією ключових активів та адмініструванням інструментів для забезпечення перевіреного резервного копіювання та відновлення системи. |
| **Планування кібербезпеки** *[Див. також KA «Безпека даних», с. 16, KA «Безпека людини», с. 44, та KA «Суспільна безпека», с. 62, щодо пов'язаного змісту.]* | Стратегічне планування | Процес визначення стратегії кібербезпеки організації — або напрямку — та визначення необхідних дій і ресурсів для реалізації такої стратегії. Ця тема охоплює такі концепції, як: визначення поточного стану організації; проведення SWOT-аналізу (сильні сторони, слабкі сторони, можливості та загрози); розроблення стратегії, що відповідає місії, цінностям і баченню організації; визначення довгострокових цілей; вибір ключових показників ефективності (KPI) для відстеження прогресу; виділення необхідного бюджету; впровадження стратегії в організації; а також щорічне оновлення та адаптація. |
| | Операційне та тактичне управління | Спроможність організації безпечно управляти організаційною технічною інфраструктурою. Ця тема включає обговорення захисту даних та конфіденційності за замовчуванням і за принципом дизайну, а також охоплює базові концепції, проблеми та техніки для ефективних і результативних операцій. Особливий акцент зроблено на вдосконаленні процесів та управлінні ланцюгом постачання. Теми включають: операційну стратегію; тактичну стратегію; проектування продуктів і послуг; аналіз і проектування процесів; планування потужностей; ощадливі виробничі системи; управління матеріалами та запасами; управління якістю та шість сигма; управління проєктами та управління ланцюгом постачання. |
| **Забезпечення безперервності бізнесу, аварійне відновлення та управління інцидентами** | | Опис ролі аварійного відновлення (DR) у межах забезпечення безперервності бізнесу (BC). Планування BC включає планування на випадок непередбачених обставин, реагування на інциденти, аварійне реагування та зусилля з резервного копіювання та відновлення організації для забезпечення доступності критичних ресурсів під час надзвичайної ситуації, тоді як аварійне відновлення стосується відновлення систем у разі катастрофи. Безперервність організацій після великих подій також є складовою. Ця тема включає створення та використання планів IR/DR/BP BC, організацію планів, нагоди для перегляду/переписування планів, вивчення знеособлених планів; студентам слід надавати можливості для написання планів на основі кейсів або реальних планів для набуття практичного досвіду. |
| | Реагування на інциденти | Реагування на інциденти (IR) стосується дій вищого керівництва для визначення процесів і процедур організації з передбачення, виявлення та пом'якшення наслідків інциденту. Ця тема включає створення та використання планів IR, організацію планів, нагоди для перегляду/переписування планів та вивчення знеособлених планів. Студентам слід надавати можливості для написання планів на основі кейсів або реальних планів для набуття практичного досвіду. |
| | Аварійне відновлення | Аварійне відновлення (DR) стосується дій вищого керівництва для визначення зусиль організації з підготовки до катастрофи та відновлення після неї. Зокрема, DR стосується відновлення систем у разі катастрофи. Ця тема включає створення та використання планів DR, організацію планів, нагоди для перегляду/переписування планів та вивчення знеособлених планів. Студентам слід надавати можливості для написання планів на основі кейсів або реальних планів для набуття практичного досвіду. |
| | Безперервність бізнесу | Безперервність бізнесу стосується дій вищого керівництва для визначення зусиль організації у разі, коли катастрофа унеможливлює використання основного місця розташування організації. Планування BC включає планування на випадок непередбачених обставин, реагування на інциденти, аварійне реагування та зусилля з резервного копіювання та відновлення організації для забезпечення доступності критичних ресурсів під час надзвичайної ситуації. Безперервність організацій після великих подій також є складовою. Навчальний зміст повинен включати створення та використання планів BC, організацію планів, нагоди для перегляду/переписування планів та вивчення знеособлених планів. Студентам слід надавати можливості для написання планів на основі кейсів або реальних планів для набуття практичного досвіду. |
| **Управління програмою безпеки** | Управління проєктами | Управління проєктами — це застосування знань, навичок, інструментів та технік до проєктної діяльності для виконання вимог проєкту. Ця тема включає: інтеграцію проєкту; управління змістом проєкту; управління часом і вартістю проєкту; управління якістю; кадрові міркування; комунікацію; управління ризиками та управління закупівлями. |
| | Управління ресурсами | Управління ресурсами — це ефективне та результативне розгортання та виділення ресурсів організації за потребою та в потрібному місці. Такі ресурси можуть включати фінансові ресурси, запаси, людські навички, виробничі ресурси або інформаційні технології. Ця тема пояснює та розвиває поточні практики управління ресурсами, зокрема в контексті проєктів, типових для кібербезпеки. |
| | Метрики безпеки | Метрики, що часто описуються як показники, є ефективними інструментами для визначення ефективності компонентів програм безпеки та спонукання до дій з їх удосконалення. Ця тема включає елементи метрик безпеки та способи їхнього проектування, розроблення, перевірки та організації. Слід включити використання метрик у різних контекстах, таких як: використання метрик безпеки в процесі прийняття рішень; використання метрик безпеки в стратегічному, тактичному та операційному плануванні; використання метрик безпеки в оцінюванні, аудиті та показниках ефективності програми безпеки. |
| | Забезпечення якості та контроль якості | Забезпечення якості (QA) та контроль якості (QC) — це методи, що використовуються для запобігання помилкам, які можуть вплинути на характеристики результату, такого як програмна система; контроль конкретно стосується методів підвищення якості цих систем. Ця тема пояснює та розвиває поточні практики QA/QC, зокрема в контексті проєктів, типових для кібербезпеки. |
| **Безпека персоналу** *[Див. також KA «Безпека людини», с. 44, щодо пов'язаного змісту.]* | Обізнаність, навчання та освіта з безпеки | Ця тема охоплює уникнення та/або належне використання страху, невизначеності та сумнівів (FUD) як інструменту підвищення обізнаності. Тема включає: фізичну безпеку; безпеку робочого місця; безпеку паролів; бездротові мережі; фішинг безпеки; обмін файлами та авторське право; безпечний серфінг; шифрування; внутрішні загрози; міжнародні подорожі; соціальні мережі та соціальну інженерію. |
| | Практики найму у сфері безпеки | Практики, що регулюються політиками та використовуються організаціями для набору, найму та навчання співробітників у всій організації. Ця тема включає принципи даної теми, а студенти повинні набути досвіду через вивчення вигаданих резюме, вигаданих перевірок анкет, вигаданих техніки інтерв'ю, результатів аналізу відбитків пальців та фінансового огляду. |
| | Практики звільнення у сфері безпеки | Практики, що регулюються політиками та використовуються організаціями для звільнення співробітників у всій організації, включаючи повернення призначених активів, видалення облікових даних та проактивне запобігання викраданню даних. Ця тема включає принципи даної теми, а студенти повинні набути досвіду через практичні завдання та симуляції. |
| | Безпека третіх сторін | Практики компаній з управління ризиками від підрядників, консультантів та персоналу ключових бізнес-партнерів. Ця тема включає принципи даної теми, а студенти повинні набути досвіду через практичні завдання та симуляції. |
| | Безпека в процесах огляду | Практики компаній з управління періодичним оглядом співробітників. Ця тема включає принципи даної теми, а студенти повинні набути досвіду через практичні завдання та симуляції. |
| | Особливі питання конфіденційності персональної інформації співробітників *[Див. також KA «Безпека даних», с. 16, KA «Безпека людини», с. 44, та KA «Суспільна безпека», с. 62, щодо пов'язаного змісту.]* | Практики компаній із захисту персональної інформації співробітників та інших зацікавлених сторін. Ця тема включає принципи даної теми, а студенти повинні набути досвіду через практичні завдання та симуляції. |
| **Операції безпеки** | Конвергенція безпеки | Злиття управлінської відповідальності в галузях корпоративної (фізичної) безпеки, корпоративного управління ризиками, комп'ютерної безпеки, мережевої безпеки та інформаційної безпеки є спостережуваним явищем на практиці в багатьох середніх та великих організаціях. Ця тема включає нові приклади конвергенції на практиці, що може бути корисним матеріалом для аудиторного обговорення нових тем. |
| | Глобальні центри операцій безпеки (GSOC) | Оптимізовані процеси можуть додавати цінність широким організаційним центрам операцій, що поєднують фізичну безпеку та кібербезпеку. Ця тема охоплює те, як співвіднесення глобальних атак із локальними заходами відповідності є необхідністю. Як атака в Малайзії впливає на бізнес-функції в Колорадо? Функції GSOC повинні мати чіткі комунікації щодо виявленої атаки, а також щодо виявленого регіону атаки та регіону її походження. GSOC повинен мати можливість повністю визначити тип атаки, її профіль і місце походження, щоб поширити цю інформацію іншим центрам операцій безпеки. |

## 4.7.2 Основні елементи та результати навчання

Від студентів вимагається демонструвати компетентність у кожному з основних понять через досягнення результатів навчання. Як правило, результати навчання лежать у межах рівнів розуміння та застосування за переглянутою таксономією Блума (http://ccecc.acm.org/assessment/blooms).

| Основні елементи | Результати навчання |
|---|---|
| Управління ризиками | Описати управління ризиками та його роль в організації. |
| | Описати техніки управління ризиками для ідентифікації та пріоритизації факторів ризику для інформаційних активів і те, як оцінюється ризик. |
| | Обговорити стратегічні варіанти, що використовуються для роботи з ризиками, та бути готовим вибирати між ними, маючи відповідну вихідну інформацію. |
| | Описати популярні методології, що використовуються в галузі для управління ризиками. |
| Управління та політика | Обговорити важливість, переваги та бажані результати управління кібербезпекою та те, як така програма може бути реалізована. |
| | Описати політику інформаційної безпеки та її роль в успішній програмі інформаційної безпеки. |
| | Описати основні типи політики інформаційної безпеки та основні компоненти кожної з них. |
| | Пояснити, що необхідно для розроблення, реалізації та підтримання ефективної політики і які наслідки може зазнати організація, якщо вона цього не робить. |
| Закони, етика та відповідність вимогам | Розмежувати закон та етику. |
| | Описати, чому кодекси етичної поведінки є важливими для фахівців із кібербезпеки та їхніх організацій. |
| | Визначити значущі національні та міжнародні закони, що стосуються кібербезпеки. |
| | Пояснити, як організації досягають відповідності національним та міжнародним законам і нормативним актам, а також конкретним галузевим стандартам. |
| Стратегія та планування | Пояснити стратегічне організаційне планування кібербезпеки та його зв'язок із загальноорганізаційним та ІТ-стратегічним плануванням. |
| | Визначити ключових організаційних зацікавлених сторін та їхні ролі. |
| | Описати основні компоненти планування реалізації системи кібербезпеки. |
