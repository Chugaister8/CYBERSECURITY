# 4.5 Галузь знань: Безпека систем

Галузь знань «Безпека систем» зосереджена на аспектах безпеки систем, що складаються з компонентів і з'єднань та використовують програмне забезпечення. Розуміння безпеки системи вимагає розгляду її не лише як набору компонентів і з'єднань, але й як цілісної одиниці самої по собі. Це потребує цілісного погляду на систему. Разом із KA «Безпека компонентів» та KA «Безпека з'єднань», KA «Безпека систем» розглядає проблеми безпеки при підключенні компонентів та їхньому використанні в складі більших систем.

Наведена нижче таблиця перелічує основні елементи, одиниці знань та теми галузі знань «Безпека систем».

**БЕЗПЕКА СИСТЕМ**

**Основні елементи:**
- Цілісний підхід,
- Політика безпеки,
- Автентифікація,
- Контроль доступу,
- Моніторинг,
- Відновлення,
- Тестування,
- Документація.

| **Управління системою** *[Див. також KA «Безпека програмного забезпечення» щодо пов'язаного змісту, с. 23.]* | | Ця одиниця знань описує техніки включення міркувань безпеки на всіх етапах управління системою. |

| | Компонування політик | Ця тема охоплює обмеженість. |
| | Використання автоматизації | Ця тема включає видобуток даних, машинне навчання та суміжні техніки, а також їхні переваги та обмеження. |
| | Встановлення виправлень та lifecycle вразливостей | Ця тема включає проблеми безпеки, що виникають при встановленні виправлень: чи потрібно встановлювати виправлення для системи, встановлення виправлень для працюючої системи, а також порядок обробки звітів про вразливості. |
| | Експлуатація | Ця тема включає безпеку в процесі експлуатації та важливість міркувань щодо зручності використання. |
| | Введення в експлуатацію та виведення з експлуатації | Ця тема описує міркування безпеки при встановленні та видаленні системи. |
| | Внутрішня загроза | Ця тема включає приклади внутрішніх загроз, таких як викрадення даних та диверсії, і охоплює контрзаходи. |
| | Документація | Ця тема охоплює документацію з безпеки та забезпечення якості, а також посібники з встановлення та керівництва користувача, зосереджені на самій системі. |
| | Системи та процедури | Ця тема розглядає процедури, що використовуються для управління системами. |
| **Доступ до системи** *[Див. також KA «Безпека людини» щодо пов'язаного змісту, с. 44.]* | | Ця одиниця знань знайомить із міркуваннями безпеки щодо контролю доступу до систем. Вона стосується ідентифікації суб'єктів та підтвердження цієї ідентифікації до бажаного рівня деталізації. Теми перетинаються з галуззю знань «Безпека людини», але тут акцент зроблено на системних елементах, а не на людських. |
| | Методи автентифікації | Методи автентифікації стосуються автентифікації людина-система або система-система; приклади включають паролі, біометрію, донгли та єдиний вхід (single sign-on). |
| | Ідентичність | Як ідентичність представлена системі? Ця тема включає ролі, а також імена тощо. |
| **Контроль системи** *[Див. також KA «Безпека даних» щодо пов'язаного змісту, с. 16.]* | | Ця одиниця знань розглядає міркування безпеки, пов'язані з контролем самої системи. Вона включає виявлення атак, компенсацію за них, захист від них та їхнє запобігання. |
| | Контроль доступу | Ця тема зосереджена на контролі доступу до ресурсів та цілісності засобів контролю, а не на контролі доступу до даних, що охоплюється галуззю знань «Безпека даних». |
| | Моделі авторизації | Ця тема охоплює управління авторизацією в багатьох системах та відмінність між автентифікацією та авторизацією. |
| | Виявлення вторгнень | Ця тема охоплює техніки на основі аномалій, неправомірного використання (на основі правил, на основі сигнатур) та специфікацій. |
| | Атаки | Ця тема охоплює моделі атак (такі як дерева та графи атак) і конкретні атаки. |
| | Засоби захисту | Ця тема включає такі приклади, як ASLR, IP-стрибки та стійкість до вторгнень. |
| | Аудит | Ця тема охоплює ведення журналів, аналіз журналів та їхній зв'язок із виявленням вторгнень. |
| | Шкідливе програмне забезпечення | Ця тема включає такі приклади, як комп'ютерні віруси, черв'яки, програми-вимагачі та інші форми шкідливого програмного забезпечення. |
| | Моделі вразливостей | Ця тема включає такі приклади, як RISOS та PA, а також переліки, такі як CVE та CWE. |
| | Тестування на проникнення | Ця тема охоплює методологію гіпотези про недоліки та інші форми (ISSAF, OSSTMM, GISTA, PTES тощо). |
| | Криміналістика *[Див. також KA «Безпека даних» щодо пов'язаного змісту, с. 16.]* | Ця тема зосереджена на системних вимогах для криміналістики. |
| | Відновлення, стійкість | Ця тема включає механізми забезпечення доступності. |
| **Виведення системи з експлуатації** | | Ця одиниця знань розглядає, як виведення системи з експлуатації до або після закінчення терміну її служби може вплинути на безпеку інших систем або організації, що використовувала систему. |
| | Деконфігурування | Ця тема розглядає, як виведення системи з експлуатації до або після закінчення терміну її служби може вплинути на безпеку інших систем або організації, що використовувала систему. Студент повинен розуміти наслідки видалення системи, або компонентів чи з'єднань у межах системи, для безпеки системи в цілому. |
| | Утилізація | Ця тема включає очищення носіїв та інші форми знищення для запобігання відновленню конфіденційної інформації (наприклад, персональних даних). |
| **Тестування системи** *[Див. також KA «Безпека програмного забезпечення», с. 23, та KA «Безпека компонентів», с. 29, щодо пов'язаного змісту.]* | | Ця одиниця знань охоплює міркування щодо тестування систем для забезпечення їхньої відповідності вимогам безпеки. |
| | Перевірка вимог | Ця тема описує методології для підтвердження того, що вимоги відповідають цілям. |
| | Перевірка компонування компонентів | Ця тема охоплює те, як тестувати систему як ціле. |
| | Модульне тестування проти системного тестування | Ця тема охоплює відмінності системного тестування від тестування компонентів і з'єднань. |
| | Формальна верифікація систем | Ця тема охоплює мови, засоби доведення теорем та ієрархічну декомпозицію. |
| **Поширені системні архітектури** *[Див. також KA «Безпека з'єднань» щодо пов'язаного змісту, с. 32.]* | | Ця одиниця знань застосовує теми цієї галузі знань до конкретних архітектур, які є або стають більш поширеними. |
| | Віртуальні машини | Ця тема охоплює гіпервізори, віртуалізацію дисків і пам'яті та використання віртуальних машин у безпеці. |
| | Промислові системи керування | Ця тема включає SCADA. |
| | Інтернет речей (IoT) | Ця тема включає такі приклади, як холодильники та датчики. |
| | Вбудовані системи | Ця тема включає такі приклади, як системи в космічних апаратах та системи, що використовуються в інших несприятливих середовищах. |
| | Мобільні системи | Ця тема включає такі приклади, як ноутбуки та смартфони. |
| | Автономні системи | Ця тема включає такі приклади, як роботи та БПЛА, що не потребують управління людиною. |
| | Системи загального призначення | Ця тема включає такі приклади, як настільні комп'ютери, ноутбуки та мейнфрейми. |

## 4.5.2 Основні елементи та результати навчання

Від студентів вимагається демонструвати компетентність у кожному з основних понять через досягнення результатів навчання. Як правило, результати навчання лежать у межах рівнів розуміння та застосування за переглянутою таксономією Блума (http://ccecc.acm.org/assessment/blooms).

| Основні елементи | Результати навчання |
|---|---|
| Цілісний підхід | Пояснити концепції довіри та надійності. |
| | Пояснити, що мається на увазі під конфіденційністю, цілісністю та доступністю. |
| | Пояснити, що таке політика безпеки та яка її роль у захисті даних і ресурсів. |
| Політика безпеки *[Див. також KA «Організаційна безпека» щодо пов'язаного змісту, с. 51.]* | Обговорити важливість політики безпеки. |
| | Пояснити, чому різні організації мають різні політики безпеки. |
| | Пояснити зв'язок між групою безпеки, конфігурацією системи та процедурами підтримання безпеки системи. |
| Автентифікація | Пояснити три властивості, що зазвичай використовуються для автентифікації. |
| | Пояснити важливість багатофакторної автентифікації. |
| | Пояснити переваги парольних фраз порівняно з паролями. |
| Контроль доступу | Описати список контролю доступу. |
| | Описати фізичний та логічний контроль доступу та порівняти їх між собою. |
| | Розмежувати авторизацію та автентифікацію. |
| Моніторинг | Обговорити, як системи виявлення вторгнень сприяють безпеці. |
| | Описати обмеження антишкідливого програмного забезпечення, такого як антивірусні програми. |
| | Обговорити варіанти використання системного моніторингу. |
| Відновлення | Пояснити, що таке стійкість, та визначити середовище, в якому вона є важливою. |
| | Обговорити основи плану аварійного відновлення. |
| | Пояснити, чому резервні копії є потенційним ризиком безпеки. |
| Тестування | Описати, що таке тестування на проникнення та чому воно є цінним. |
| | Обговорити, як документувати тест, що виявляє вразливість. |
| | Обговорити важливість перевірки вимог. |
| Документація | Обговорити важливість документування правильного встановлення та конфігурування системи. |
| | Вміти складати документацію з вторгнень на хості та в мережі. |
| | Вміти пояснювати наслідки для безпеки нечіткої або неповної документації щодо роботи системи. |
