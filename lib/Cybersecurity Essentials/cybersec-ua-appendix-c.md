# Додаток C. Попередні рамкові засади кібербезпеки NIST

Рамковий документ, розроблений у співпраці з промисловістю, надає організаціям настанови щодо управління ризиками кібербезпеки. Ключова мета рамкового документа — спонукати організації розглядати ризик кібербезпеки як пріоритет, аналогічний фінансовому, безпековому та операційному ризику, з урахуванням більших системних ризиків, притаманних критичній інфраструктурі.

Рамковий документ спирається на чинні стандарти, настанови та найкращі практики для досягнення результатів, що можуть допомогти організаціям управляти ризиками кібербезпеки. Покладаючись на практики, розроблені, кероровані та оновлювані промисловістю, рамковий документ еволюціонуватиме разом із технологічними досягненнями та бізнес-вимогами.

**РИСУНОК C.1.** Зацікавлені сторони рамкового документа NIST

## ІДЕНТИФІКАЦІЯ (IDENTIFY)

### Управління активами (ID.AM)

**Методологія:** Дані, персонал, пристрої, системи та об'єкти, що дозволяють організації досягати бізнес-цілей, ідентифікуються та управляються відповідно до їхньої відносної важливості для бізнес-цілей та стратегії управління ризиками організації.

| Підкатегорія | Опис |
|---|---|
| ID.AM-1 | Фізичні пристрої та системи в організації інвентаризовано. |
| ID.AM-2 | Програмні платформи та застосунки в організації інвентаризовано. |
| ID.AM-3 | Організаційні комунікації та потоки даних нанесено на карту. |
| ID.AM-4 | Зовнішні інформаційні системи каталогізовано. |
| ID.AM-5 | Ресурси (апаратні пристрої, дані, програмне забезпечення) пріоритизовано за класифікацією, критичністю та бізнес-цінністю. |
| ID.AM-6 | Ролі та обов'язки з кібербезпеки для всього персоналу та сторонніх зацікавлених сторін (постачальників, клієнтів, партнерів) встановлено. |

### Бізнес-середовище (ID.BE)

**Методологія:** Місія, цілі, зацікавлені сторони та діяльність організації зрозумілі та визначені за пріоритетністю; ця інформація використовується для визначення ролей, обов'язків та функцій управління ризиками кібербезпеки.

| Підкатегорія | Опис |
|---|---|
| ID.BE-1 | Роль організації в ланцюгу постачання ідентифікована та комунікована. |
| ID.BE-2 | Місце організації в критичній інфраструктурі та її галузевому секторі ідентифіковано та комунікований. |
| ID.BE-3 | Місія, цілі та діяльність організації встановлені та комунікована. |
| ID.BE-4 | Залежності та критичні функції для надання критичних послуг встановлено. |
| ID.BE-5 | Вимоги до стійкості для забезпечення надання критичних послуг встановлено. |

### Управління (ID.GV)

**Методологія:** Політики, процедури та процеси для управління та моніторингу нормативно-правових, ризикових, екологічних та операційних вимог організації зрозумілі та інформують управління ризиками кібербезпеки.

| Підкатегорія | Опис |
|---|---|
| ID.GV-1 | Організаційну політику інформаційної безпеки встановлено. |
| ID.GV-2 | Ролі та обов'язки з інформаційної безпеки узгоджені та скоординовані з внутрішніми ролями та зовнішніми партнерами. |
| ID.GV-3 | Правові та нормативні вимоги щодо кібербезпеки, включно з конфіденційністю та зобов'язаннями щодо громадянських свобод, зрозумілі та виконуються. |
| ID.GV-4 | Урядові процеси та процеси управління ризиками враховують ризики кібербезпеки. |

### Оцінка ризиків (ID.RA)

**Методологія:** Організація розуміє ризик кібербезпеки для організаційних операцій (включно з місією, функціями, іміджем або репутацією), організаційними активами та особами.

| Підкатегорія | Опис |
|---|---|
| ID.RA-1 | Вразливості активів ідентифіковані та задокументовані. |
| ID.RA-2 | Інформація про загрози та вразливості отримується з форумів та джерел обміну інформацією. |
| ID.RA-3 | Загрози, як внутрішні, так і зовнішні, ідентифіковані та задокументовані. |
| ID.RA-4 | Потенційні бізнес-впливи та ймовірності ідентифіковані. |
| ID.RA-5 | Загрози, вразливості, ймовірності та впливи використовуються для визначення ризику. |
| ID.RA-6 | Відповіді на ризики ідентифіковані та пріоритизовані. |

### Стратегія управління ризиками (ID.RM)

**Методологія:** Пріоритети, обмеження, допустимий рівень ризику та припущення організації встановлені та використовуються для прийняття операційних рішень щодо ризику.

| Підкатегорія | Опис |
|---|---|
| ID.RM-1 | Процеси управління ризиками встановлені, управляються та узгоджені з організаційними зацікавленими сторонами. |
| ID.RM-2 | Організаційна допустимість ризику визначена та чітко виражена. |
| ID.RM-3 | Визначення організаційної допустимості ризику ґрунтується на ролі в критичній інфраструктурі та галузевому аналізі ризиків. |

## ЗАХИСТ (PROTECT)

### Контроль доступу (PR.AC)

**Методологія:** Доступ до активів та пов'язаних об'єктів обмежений авторизованими користувачами, процесами або пристроями та авторизованими діями та транзакціями.

| Підкатегорія | Опис |
|---|---|
| PR.AC-1 | Особисті дані та облікові дані управляються для авторизованих пристроїв та користувачів. |
| PR.AC-2 | Фізичний доступ до активів управляється та захищається. |
| PR.AC-3 | Дистанційний доступ управляється. |
| PR.AC-4 | Дозволи на доступ управляються з урахуванням принципів мінімальних привілеїв та розподілу обов'язків. |
| PR.AC-5 | Цілісність мережі захищена, включно з мережевою сегментацією де це доцільно. |

### Обізнаність та навчання (PR.AT)

**Методологія:** Персонал та партнери організації отримують освіту з питань обізнаності в кібербезпеці та належним чином навчені для виконання своїх обов'язків з інформаційної безпеки.

| Підкатегорія | Опис |
|---|---|
| PR.AT-1 | Всі користувачі поінформовані та навчені. |
| PR.AT-2 | Привілейовані користувачі розуміють ролі та обов'язки. |
| PR.AT-3 | Сторонні зацікавлені сторони (постачальники, клієнти, партнери) розуміють ролі та обов'язки. |
| PR.AT-4 | Старше керівництво розуміє ролі та обов'язки. |
| PR.AT-5 | Персонал фізичної та інформаційної безпеки розуміє ролі та обов'язки. |

### Безпека даних (PR.DS)

**Методологія:** Інформація та записи (дані) управляються відповідно до стратегії управління ризиками організації для захисту конфіденційності, цілісності та доступності інформації.

| Підкатегорія | Опис |
|---|---|
| PR.DS-1 | Дані у стані спокою захищені. |
| PR.DS-2 | Дані у русі захищені. |
| PR.DS-3 | Активи формально управляються протягом видалення, передачі та розпорядження. |
| PR.DS-4 | Підтримується достатня потужність для забезпечення доступності. |
| PR.DS-5 | Впроваджено захист від витоків даних. |
| PR.DS-6 | Механізми перевірки цілісності використовуються для верифікації цілісності програмного забезпечення, мікропрограми та інформації. |
| PR.DS-7 | Середовище(а) розробки та тестування відокремлені від виробничого середовища. |

### Процеси та процедури захисту інформації (PR.IP)

**Методологія:** Політики безпеки (що охоплюють мету, сферу, ролі, обов'язки, управлінські зобов'язання та координацію між організаційними суб'єктами), процеси та процедури підтримуються та використовуються для управління захистом інформаційних систем та активів.

| Підкатегорія | Опис |
|---|---|
| PR.IP-1 | Базову конфігурацію інформаційно-технологічних/промислових систем управління створено та підтримується. |
| PR.IP-2 | Для управління системами реалізовано Життєвий цикл розробки системи (SDLC). |
| PR.IP-3 | Впроваджено процеси контролю змін конфігурації. |
| PR.IP-4 | Резервні копії інформації виконуються, підтримуються та періодично тестуються. |
| PR.IP-5 | Виконуються вимоги щодо політики та нормативних актів щодо фізичного операційного середовища організаційних активів. |
| PR.IP-6 | Дані знищуються відповідно до політики. |
| PR.IP-7 | Процеси захисту постійно вдосконалюються. |
| PR.IP-8 | Ефективністю захисних технологій діляться з відповідними сторонами. |
| PR.IP-9 | Плани реагування (Реагування на інциденти та Забезпечення безперервності бізнесу) та плани відновлення (Відновлення після інцидентів та катастроф) наявні та управляються. |
| PR.IP-10 | Плани реагування та відновлення тестуються. |
| PR.IP-11 | Кібербезпека включена в практики управління персоналом (деавторизація, перевірка персоналу). |
| PR.IP-12 | Розроблено та реалізовано план управління вразливостями. |

### Обслуговування (PR.MA)

**Методологія:** Технічне обслуговування та ремонт компонентів промислових систем управління та інформаційних систем виконуються відповідно до політик та процедур.

| Підкатегорія | Опис |
|---|---|
| PR.MA-1 | Технічне обслуговування та ремонт організаційних активів виконується та реєструється своєчасно із затвердженими та контрольованими інструментами. |
| PR.MA-2 | Дистанційне технічне обслуговування організаційних активів затверджується, реєструється та виконується способом, що запобігає несанкціонованому доступу. |

### Захисна технологія (PR.PT)

**Методологія:** Технічні рішення безпеки управляються для забезпечення безпеки та стійкості систем та активів відповідно до пов'язаних політик, процедур і угод.

| Підкатегорія | Опис |
|---|---|
| PR.PT-1 | Записи аудиту/журналів визначені, задокументовані, реалізовані та переглядаються відповідно до політики. |
| PR.PT-2 | Знімні носії захищені та їхнє використання обмежене відповідно до політики. |
| PR.PT-3 | Доступ до систем та активів контролюється з урахуванням принципу мінімальної функціональності. |
| PR.PT-4 | Мережі зв'язку та управління захищені. |

## ВИЯВЛЕННЯ (DETECT)

### Аномалії та події (DE.AE)

**Методологія:** Аномальна активність виявляється своєчасно та потенційний вплив подій зрозумілий.

| Підкатегорія | Опис |
|---|---|
| DE.AE-1 | База мережевих операцій та очікуваних потоків даних для користувачів та систем встановлена та управляється. |
| DE.AE-2 | Виявлені події аналізуються для розуміння цілей та методів атаки. |
| DE.AE-3 | Дані про події агрегуються та корелюються з кількох джерел та сенсорів. |
| DE.AE-4 | Вплив подій визначається. |
| DE.AE-5 | Порогові значення сповіщень про інциденти встановлені. |

### Постійний моніторинг безпеки (DE.CM)

**Методологія:** Інформаційна система та активи моніторяться з дискретними інтервалами для виявлення подій кібербезпеки та перевірки ефективності захисних заходів.

| Підкатегорія | Опис |
|---|---|
| DE.CM-1 | Мережа моніториться для виявлення потенційних подій кібербезпеки. |
| DE.CM-2 | Фізичне середовище моніториться для виявлення потенційних подій кібербезпеки. |
| DE.CM-3 | Діяльність персоналу моніториться для виявлення потенційних подій кібербезпеки. |
| DE.CM-4 | Шкідливий код виявляється. |
| DE.CM-5 | Несанкціонований мобільний код виявляється. |
| DE.CM-6 | Діяльність зовнішнього постачальника послуг моніториться для виявлення потенційних подій кібербезпеки. |
| DE.CM-7 | Здійснюється моніторинг несанкціонованого персоналу, підключень, пристроїв та програмного забезпечення. |
| DE.CM-8 | Виконується сканування вразливостей. |

### Процеси виявлення (DE.DP)

**Методологія:** Процеси та процедури виявлення підтримуються та тестуються для забезпечення своєчасної та адекватної обізнаності про аномальні події.

| Підкатегорія | Опис |
|---|---|
| DE.DP-1 | Ролі та обов'язки з виявлення чітко визначені для забезпечення підзвітності. |
| DE.DP-2 | Діяльність з виявлення відповідає всім застосовним вимогам. |
| DE.DP-3 | Процеси виявлення тестуються. |
| DE.DP-4 | Інформація про виявлення подій комунікується відповідним сторонам. |
| DE.DP-5 | Процеси виявлення постійно вдосконалюються. |

## РЕАГУВАННЯ (RESPOND)

### Планування реагування (RS.RP)

**Методологія:** Процеси та процедури реагування виконуються та підтримуються для забезпечення своєчасного реагування на виявлені події кібербезпеки.

| Підкатегорія | Опис |
|---|---|
| RS.RP-1 | План реагування виконується під час або після події. |

### Комунікації (RS.CO)

**Методологія:** Заходи з реагування координуються з внутрішніми та зовнішніми зацікавленими сторонами, включно із зовнішньою підтримкою від правоохоронних органів.

| Підкатегорія | Опис |
|---|---|
| RS.CO-1 | Персонал знає свої ролі та порядок дій при необхідності реагування. |
| RS.CO-2 | Події повідомляються відповідно до встановлених критеріїв. |
| RS.CO-3 | Інформація передається відповідно до планів реагування. |
| RS.CO-4 | Координація з зацікавленими сторонами здійснюється відповідно до планів реагування. |
| RS.CO-5 | Добровільний обмін інформацією здійснюється із зовнішніми зацікавленими сторонами для підвищення загальної ситуаційної обізнаності в кібербезпеці. |

### Аналіз (RS.AN)

**Методологія:** Аналіз проводиться для забезпечення адекватного реагування та підтримки діяльності з відновлення.

| Підкатегорія | Опис |
|---|---|
| RS.AN-1 | Сповіщення від систем виявлення розслідуються. |
| RS.AN-2 | Вплив інциденту зрозумілий. |
| RS.AN-3 | Виконується судово-технічна експертиза. |
| RS.AN-4 | Інциденти класифікуються відповідно до планів реагування. |

### Пом'якшення (RS.MI)

**Методологія:** Виконуються заходи для запобігання розширенню події, пом'якшення її наслідків та ліквідації інциденту.

| Підкатегорія | Опис |
|---|---|
| RS.MI-1 | Інциденти стримуються. |
| RS.MI-2 | Інциденти пом'якшуються. |
| RS.MI-3 | Нещодавно виявлені вразливості пом'якшуються або документуються як прийнятні ризики. |

### Вдосконалення (RS.IM)

**Методологія:** Організаційна діяльність з реагування вдосконалюється шляхом включення уроків, засвоєних із поточної та попередньої діяльності з виявлення/реагування.

| Підкатегорія | Опис |
|---|---|
| RS.IM-1 | Плани реагування включають засвоєні уроки. |
| RS.IM-2 | Стратегії реагування оновлюються. |

## ВІДНОВЛЕННЯ (RECOVER)

### Планування відновлення (RC.RP)

**Методологія:** Процеси та процедури відновлення виконуються та підтримуються для забезпечення своєчасного відновлення систем або активів, що постраждали від подій кібербезпеки.

| Підкатегорія | Опис |
|---|---|
| RC.RP-1 | План відновлення виконується під час або після події. |

### Вдосконалення (RC.IM)

**Методологія:** Планування та процеси відновлення вдосконалюються шляхом включення засвоєних уроків у майбутню діяльність.

| Підкатегорія | Опис |
|---|---|
| RC.IM-1 | Плани відновлення включають засвоєні уроки. |
| RC.IM-2 | Стратегії відновлення оновлюються. |

### Комунікації (RC.CO)

**Методологія:** Діяльність з відновлення координується з внутрішніми та зовнішніми сторонами: координаційними центрами, Інтернет-провайдерами, власниками атакуючих систем, жертвами, іншими CSIRT та постачальниками.

| Підкатегорія | Опис |
|---|---|
| RC.CO-1 | Управляються зв'язки з громадськістю. |
| RC.CO-2 | Репутація після події відновлюється. |
| RC.CO-3 | Діяльність з відновлення комунікується внутрішнім зацікавленим сторонам, виконавчому керівництву та управлінським командам. |

## Підсумок функцій рамкового документа NIST

| Функція | Мета |
|---|---|
| **Ідентифікація** | Розробити організаційне розуміння для управління ризиком кібербезпеки для систем, активів, даних та можливостей |
| **Захист** | Розробити та реалізувати відповідні заходи захисту для забезпечення надання послуг критичної інфраструктури |
| **Виявлення** | Розробити та реалізувати відповідні заходи для виявлення подій кібербезпеки |
| **Реагування** | Розробити та реалізувати відповідні заходи для вжиття дій щодо виявленої події кібербезпеки |
| **Відновлення** | Розробити та реалізувати відповідні заходи для підтримки планів стійкості та відновлення будь-яких можливостей або послуг, що постраждали внаслідок події кібербезпеки |
