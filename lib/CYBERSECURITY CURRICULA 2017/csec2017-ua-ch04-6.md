# 4.6 Галузь знань: Безпека людини

Галузь знань «Безпека людини» зосереджена на захисті даних та конфіденційності людей в контексті організацій (тобто як співробітників) та особистого життя, а також на вивченні поведінки людей стосовно кібербезпеки.

## 4.6.1 Одиниці знань та теми

Люди несуть відповідальність за забезпечення конфіденційності, цілісності та доступності (CIA) своїх організаційних та особистих комп'ютерних систем, а ця відповідальність залежить від кожної з одиниць знань «Безпека людини», наведених нижче. Наступна таблиця перелічує основні елементи, одиниці знань та теми галузі знань «Безпека людини».

---

**БЕЗПЕКА ЛЮДИНИ**

**Основні елементи:**
- Управління ідентичністю,
- Соціальна інженерія,
- Обізнаність та розуміння,
- Конфіденційність та безпека соціальної поведінки,
- Конфіденційність та безпека персональних даних.

---

| Одиниці знань | Теми | Опис / Навчальні рекомендації |
|---|---|---|
| **Управління ідентичністю** | Ідентифікація та автентифікація людей і пристроїв | Ця тема надає огляд різних методів контролю доступу для демонстрації переваг та труднощів кожного з них. Теми включають: огляд контролю мережевого доступу (NAC), управління ідентичністю та доступом (IAM), ролі, системи ідентифікації та автентифікації на основі кількох методів, біометричні системи автентифікації (включаючи такі питання, як точність/FAR/FRR, стійкість, конфіденційність тощо), а також зручність використання та прийнятність методів. |
| | Контроль фізичних та логічних активів | Ця тема охоплює різні засоби контролю доступу до фізичних активів, включаючи апаратне забезпечення системи, мережеві активи, пристрої резервного копіювання/зберігання даних тощо. Приклади включають: NAC, IAM, контроль доступу на основі правил (RAC), контроль доступу на основі ролей (RBAC), методи відстеження інвентаризації та методи створення ідентичності (який тип ідентифікатора користувача допомагає підвищити безпеку при контролі доступу, наприклад, abc1234, ім'я та прізвище, перша буква імені та прізвище). |
| | Ідентичність як послуга (IaaS) | Ця тема охоплює управління ідентичністю як послугою (наприклад, хмарна ідентичність) і порушує такі питання, як: система перебуває поза контролем користувача, без можливості дізнатися, що відбулося з інформацією в системі, аудит доступу, забезпечення відповідності та гнучкість для швидкого відкликання прав доступу. |
| | Сторонні сервіси ідентичності | Ця тема надає огляд інфраструктури автентифікації, що використовується для побудови, розміщення та управління сторонніми сервісами ідентичності. Теми включають: локальні, хмарні, централізовані сервіси ідентичності/інструменти управління паролями, управління привілеями кінцевих точок тощо. |
| | Атаки на контроль доступу та заходи пом'якшення | Ця тема надає огляд різних типів атак на контроль доступу з метою крадіжки даних або облікових даних користувачів, а також заходів пом'якшення для їхньої протидії. Теми включають: атаки на паролі, словникові атаки, атаки грубої сили та спуфінг-атаки; багатофакторну автентифікацію; суворі парольні політики; захищені файли паролів; обмеження доступу до систем тощо. |
| **Соціальна інженерія** | Типи атак соціальної інженерії | Ця тема надає огляд різних способів, якими кіберзлочинці або зловмисні групи експлуатують слабкі місця в організаціях, системах, мережах та особистій інформації для забезпечення подальшої кібератаки. Запропоновані теми включають: фішинг та цільовий фішинг, фізичне втручання/видавання себе за іншу особу, вішинг (телефонний фішинг), компрометацію ділової електронної пошти та підкидання приманки. |
| | Психологія атак соціальної інженерії | Ця тема надає огляд психологічних та поведінкових факторів, пов'язаних із тим, чому люди піддаються атакам соціальної інженерії. Запропоновані теми включають: мислення з позиції супротивника, вплив емоційних реакцій на прийняття рішень, когнітивні упередження щодо ризиків та вигод, а також довіру. |
| | Введення користувачів в оману | Ця тема надає огляд інтерфейсів систем обміну повідомленнями та браузерів та/або взаємодії з користувачем, що можуть бути використані для введення користувачів в оману. Запропоновані теми включають: підробку відправників повідомлень, оманливі URL-адреси, те, як користувачі оцінюють та довіряють вебсторінкам та електронним листам, а також поведінку користувачів під час фішингових та інших браузерних попереджень. |
| | Виявлення та пом'якшення атак соціальної інженерії | Ця тема надає сценарні практичні завдання через симуляцію або віртуальні інструменти для створення середовища різних атак соціальної інженерії. Практичний досвід використання інструментів та технічних підходів для виявлення та/або пом'якшення різних загроз соціальної інженерії. Запропоновані інструменти: фільтрація електронної пошти, чорні списки, інструменти управління інформацією про безпеку та подіями (SIEM), IDS/IPS. |
| | Зловживання системою та неналежна поведінка користувача | Ця тема надає огляд навмисного та ненавмисного зловживання системою, кібербулінгу, кіберхакінгу, наївної поведінки та етичних дилем, пов'язаних із рішеннями щодо безпеки системи. |
| **Особиста відповідність правилам/політиці/етичним нормам кібербезпеки** *[Див. також KA «Суспільна безпека» щодо пов'язаного змісту, с. 62.]* | Застосування та правила поведінки | Ця тема надає огляд методів та технік спонукання людей дотримуватися правил/політик/етичних норм. Теми включають: наслідки за недотримання правил/політик/етичних норм кібербезпеки, документацію та контрольний журнал (докази відповідності для підтвердження дотримання правил/політик/етичних норм кібербезпеки), а також знання про відповідальність за недотримання правил безпеки/політики/етичних норм. Стимули для збереження роботи (особливо після навчання та підготовки щодо належних правил/політики/етичних норм особи несуть юридичну відповідальність за їх недотримання як співробітники), а також особи можуть втратити ідентичність/доступ в особистому житті через недотримання. |
| | Належна поведінка в умовах невизначеності | Ця тема надає огляд методів та технік, яких слід дотримуватися в ситуаціях невизначеності щодо того, як реагувати на ситуацію з кібербезпеки. Теми включають: CyberIQ, інтелектуальну адаптивність, критичне мислення, розуміння правильних і неправильних виборів, прийняття рішень в умовах невизначеності, раціональне та ірраціональне мислення, етичне мислення/рішення та поведінку за відсутності чіткого процесу (звітування/контактна особа тощо), а також пом'якшення людської помилки. |
| | Сприйняття ризику та комунікація | Ця тема охоплює те, як користувачі сприймають ризики кібербезпеки та реагують на них, когнітивні упередження при оцінюванні ризиків, метафори для комунікації конкретних ризиків безпеки та те, як формулювати повідомлення щодо ризиків. Визначення ментальної моделі, вплив ментальних моделей на поведінку користувача, а також поширені ментальні моделі (народні моделі) кібербезпеки та конфіденційності. |
| **Обізнаність та розуміння** *[Див. також KA «Організаційна безпека» щодо пов'язаного змісту, с. 51.]* | Кібергігієна | Ця тема надає обговорення та завдання, зосереджені на індивідуальних обов'язках (а не організації) для захисту від кіберзагроз та кібератак та їхнього пом'якшення. Теми включають: створення паролів, зберігання паролів, інструменти пом'якшення (наприклад, антивірусне програмне забезпечення), способи ідентифікації безпечних вебсайтів, визначення рівнів налаштувань конфіденційності тощо. |
| | Навчання користувачів кібербезпеці | Методи навчання кінцевих користувачів щодо різних загроз кібербезпеці/конфіденційності та відповідних форм поведінки. Теми включають: методи підвищення обізнаності користувачів (дошкільна освіта–12 клас, співробітники, суспільство тощо), методи надання освіти та навчання з кібербезпеки (наприклад, плакати, буклети, комп'ютерне навчання, гейміфікація, стилі комунікації, оформлення повідомлень, способи охоплення різних аудиторій і спільнот користувачів, особи з обмеженими можливостями та/або когнітивними порушеннями), час та підкріплення навчання, а також вплив навчання на знання та поведінку користувачів. |
| | Обізнаність про кіберуразливості та кіберзагрози | Ця тема надає огляд загроз, з якими стикаються кінцеві користувачі, а також страху, невизначеності та сумнівів (FUD). Запропоновані теми включають: попереджувальні ознаки внутрішніх вразливостей та загроз з боку співробітників, обізнаність про крадіжку ідентичності, компрометацію ділової електронної пошти, загрозу безкоштовних/відкритих мереж Wi-Fi, а також шкідливе програмне забезпечення, шпигунське програмне забезпечення та програми-вимагачі. |
| **Соціальна та поведінкова конфіденційність** *[Див. також KA «Суспільна безпека» щодо пов'язаного змісту, с. 62.]* | Соціальні теорії конфіденційності | Ця тема надає огляд різних теорій конфіденційності з соціальної психології та соціальних наук, акцентуючи увагу на конфіденційності, що передбачає взаємодію з іншими людьми, а не організаціями. Запропоновані теми включають: компроміси та ризики конфіденційності в соціальному контексті, контроль та обізнаність щодо згоди на обробку даних, моніторинг персональної інформації, регуляторний захист та проблеми підтримання соціальної конфіденційності. |
| | Конфіденційність та безпека в соціальних мережах | Ця тема надає огляд поведінки та проблем конфіденційності користувачів при захисті персональної інформації під час використання соціальних мереж. Запропоновані теми включають: рішення користувачів щодо розкриття інформації в мережі та відповідні форми поведінки, персони та управління ідентичністю, визначення аудиторії та засоби контролю соціального доступу, інтерфейсні та захисні механізми управління конфіденційністю на різних платформах соціальних мереж, труднощі управління часовими межами, а також особисті/робочі межі соціальних мереж. |
| **Конфіденційність та безпека персональних даних** *[Див. також KA «Безпека даних», с. 16, та KA «Організаційна безпека», с. 51, щодо пов'язаного змісту.]* | Чутливі персональні дані (SPD) | Ця тема надає огляд типів персональних даних (PD), включаючи персональні дані, що ідентифікують особу (PII), які є особливо чутливими через ризик того, що така інформація може бути використана неналежним чином на значну шкоду особі у фінансовому, трудовому або соціальному плані. Запропоновані теми включають: приклади елементів чутливих персональних даних (SPD) (номер соціального страхування, номер соціального страхування або інший ідентифікаційний номер, виданий державою, наприклад, номер посвідчення водія або паспорта; номер банківського рахунку; номери кредитних карток; медична інформація; біометричні або генетичні дані тощо), правила, що регулюють збір, використання та розповсюдження SPD, а також можливості виведення SPD. |
| | Особисте відстеження та цифровий слід | Відстеження місцезнаходження, відстеження вебтрафіку, мережеве відстеження, відстеження особистих пристроїв, записи цифрових асистентів (Siri, Alexa тощо). Теми включають: поведінку та проблеми користувачів щодо кожного з цих видів відстеження, а також сучасні методи обмеження відстеження та захисту конфіденційності. |
| | Зручність використання та користувацький досвід | Визначення зручності використання та користувацького досвіду, а також вплив зручності використання (або її відсутності) на безпеку та конфіденційність системи. Теми включають: приклади проблем зручності використання в традиційних системах безпеки, таких як автентифікація або шифрування, компроміси між зручністю використання та безпекою в системах, методи оцінювання зручності використання систем безпеки та конфіденційності. |
| | Людські фактори безпеки | Студенти зможуть працювати на перетині людських факторів, інформатики та галузі забезпечення якості. Це повинно включати сильне ядро обчислень та поглиблені людські фактори та забезпечення якості. Теми включають: прикладну психологію в контексті мислення з позиції супротивника та політик безпеки, економіку безпеки, регуляторні середовища, відповідальність, зобов'язання, самовизначення, видавання себе за іншу особу та шахрайство (наприклад, фішинг та цільовий фішинг, довіра, обман, опір біометричній автентифікації та управлінню ідентичністю). |
| **Зручна безпека та конфіденційність** *[Див. також KA «Організаційна безпека», с. 51, та KA «Суспільна безпека», с. 62, щодо пов'язаного змісту.]* | Обізнаність щодо політик та їхнє розуміння | Ця тема надає огляд регуляторних політик (наприклад, HIPAA, FERPA, PII) та методів або технік, яких слід дотримуватися при виникненні ситуації, пов'язаної з безпекою. Теми включають: навчання для оновлення знань щодо змін у політиках, повторний розгляд існуючих загроз та тестування знань для розуміння політики у сфері захисту даних. Через збіг тем слід також посилатися на одиниці знань у галузях знань «Суспільна безпека» та «Організаційна безпека». |
| | Політика конфіденційності | Ця тема надає огляд політик конфіденційності в соціальних та локалізованих варіантах. Слід розглянути юрисдикційні відмінності у визначеннях політики конфіденційності. Відносини між особистими, організаційними або державними політиками конфіденційності також повинні розглядатися з точки зору користувача. Додаткові теми повинні включати: вплив політики конфіденційності на нові інструменти/програмне забезпечення, визначення потреби в інструментах та техніках для охоплення в більшості галузей. Крім того, необхідно забезпечити сповіщення користувачів про те, як використовуються їхні дані, щоб вони могли зробити обґрунтований вибір щодо надання своєї інформації. |
| | Рекомендації щодо проектування та їхні наслідки | Рекомендації включають: зменшення навантаження та кількості рішень для користувача, надання безпечних налаштувань за замовчуванням, зменшення ненавмисних помилок безпеки та конфіденційності, надання загрозам і ризикам контекстуального та конкретного характеру, а також зменшення технічної мови та жаргону. |

## 4.6.2 Основні елементи та результати навчання

Від студентів вимагається демонструвати компетентність у кожному з основних понять через досягнення результатів навчання. Як правило, результати навчання лежать у межах рівнів розуміння та застосування за переглянутою таксономією Блума (http://ccecc.acm.org/assessment/blooms).

| Основні елементи | Результати навчання |
|---|---|
| Управління ідентичністю | Пояснити різницю між ідентифікацією, автентифікацією та авторизацією доступу людей і пристроїв. |
| | Обговорити важливість контрольних журналів та ведення журналів для ідентифікації та автентифікації. |
| | Продемонструвати здатність реалізовувати концепцію найменших привілеїв та розмежування обов'язків. |
| | Продемонструвати загальне розуміння атак на контроль доступу та заходів їхнього пом'якшення. |
| Соціальна інженерія | Продемонструвати загальне розуміння типів атак соціальної інженерії, психології атак соціальної інженерії та введення користувачів в оману. |
| | Продемонструвати здатність ідентифікувати типи атак соціальної інженерії. |
| | Продемонструвати здатність реалізовувати підходи для виявлення та пом'якшення атак соціальної інженерії. |
| Обізнаність та розуміння | Обговорити важливість кібергігієни, навчання користувачів кібербезпеці, а також обізнаності про кіберуразливості та кіберзагрози. |
| | Описати основні теми в межах програм навчання, підготовки та підвищення обізнаності з безпеки (SETA). |
| | Обговорити важливість SETA як контрзаходів. |
| | Обговорити важливість сприйняття ризику та комунікації в контексті ментальних моделей кібербезпеки та конфіденційності. |
| Конфіденційність та безпека соціальної поведінки | Порівняти та зіставити різні теорії конфіденційності з соціальної психології та соціальних наук. |
| | Описати концепції компромісів і ризиків конфіденційності в соціальному контексті, контролю та обізнаності щодо згоди на обробку даних, моніторингу персональної інформації, регуляторного захисту та проблем підтримання соціальної конфіденційності. |
| | Обговорити важливість конфіденційності та безпеки в соціальних мережах. |
| Конфіденційність та безпека персональних даних | Обговорити важливість захисту чутливих персональних даних (SPD) та персональних даних, що ідентифікують особу (PII). |
| | Обговорити важливість правил, що регулюють збір, використання та розповсюдження SPD, а також можливості виведення SPD. |
| | Описати концепції особистого відстеження та цифрового сліду, розуміючи рівень втручання таких інструментів у конфіденційність. |
